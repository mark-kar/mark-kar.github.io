Добро пожаловать в мой первый пост в блоге! В этой статье я проведу вас через основы настройки персонального веб-сайта с помощью Jekyll и GitHub Pages.

## Почему Jekyll?

Jekyll — это простой статический генератор сайтов с поддержкой блога, который идеально подходит для персональных веб-сайтов, блогов и страниц проектов. Вот несколько причин, почему я выбрал его:

- **Простота**: Не требуется баз данных или сложной настройки
- **Скорость**: Генерирует статические HTML-файлы, которые быстро загружаются
- **Интеграция с GitHub**: Бесшовно работает с GitHub Pages
- **Поддержка Markdown**: Пишите контент в легко читаемом формате Markdown

## Начало работы

### Предварительные требования

Перед началом убедитесь, что у вас установлено следующее:

1. Ruby (версия 2.5.0 или выше)
2. RubyGems
3. GCC и Make (для установки нативных расширений)

### Установка

Чтобы установить Jekyll и Bundler, выполните:

```bash
gem install bundler jekyll
```

### Создание вашего сайта

Создайте новый сайт Jekyll:

```bash
jekyll new my-awesome-site
cd my-awesome-site
```

Затем соберите и запустите ваш сайт локально:

```bash
bundle exec jekyll serve
```

Теперь вы можете просмотреть ваш сайт по адресу `http://localhost:4000`.

## Размещение на GitHub Pages

GitHub Pages невероятно упрощает бесплатное размещение вашего сайта Jekyll:

1. Создайте новый репозиторий на GitHub с именем `yourusername.github.io`
2. Отправьте ваш сайт Jekyll в этот репозиторий
3. Включите GitHub Pages в настройках репозитория
4. Ваш сайт будет доступен по адресу `https://yourusername.github.io`

## Написание постов

Все сообщения блога размещаются в каталоге `_posts`. Формат имени файла важен:

```
ГОД-МЕСЯЦ-ДЕНЬ-заголовок.РАЗМЕТКА
```

Например: `2024-01-15-getting-started.md`

Каждый пост начинается с front matter:

```yaml
---
layout: post
title: "Мой первый пост"
date: 2024-01-15
categories: [tutorial]
---
```
