Добро пожаловать в мой первый пост в блоге! В этой статье я расскажу вам об основах настройки персонального веб-сайта с использованием Jekyll и GitHub Pages.

## Почему Jekyll?

Jekyll - это простой генератор статических сайтов с поддержкой блога, который идеально подходит для персональных веб-сайтов, блогов и страниц проектов. Вот несколько причин, почему я выбрал его:

- **Простота**: Не требует баз данных или сложной настройки
- **Скорость**: Генерирует статические HTML-файлы, которые быстро загружаются
- **Интеграция с GitHub**: Бесшовно работает с GitHub Pages
- **Поддержка Markdown**: Пишите контент в легко читаемом формате Markdown

## Начало работы

### Предварительные требования

Перед началом убедитесь, что у вас установлены следующие компоненты:

1. Ruby (версия 2.5.0 или выше)
2. RubyGems
3. GCC и Make (для установки нативных расширений)

### Установка

Для установки Jekyll и Bundler выполните:

```bash
gem install bundler jekyll
```

### Создание вашего сайта

Создайте новый сайт на Jekyll:

```bash
jekyll new my-awesome-site
cd my-awesome-site
```

Затем соберите и запустите ваш сайт локально:

```bash
bundle exec jekyll serve
```

Теперь вы можете просмотреть свой сайт по адресу `http://localhost:4000`.

## Развертывание на GitHub Pages

GitHub Pages невероятно упрощает хостинг вашего сайта на Jekyll бесплатно:

1. Создайте новый репозиторий на GitHub с именем `yourusername.github.io`
2. Отправьте ваш сайт на Jekyll в этот репозиторий
3. Включите GitHub Pages в настройках вашего репозитория
4. Ваш сайт будет доступен по адресу `https://yourusername.github.io`

## Написание постов

Все сообщения блога размещаются в директории `_posts`. Формат имени файла важен:

```
YEAR-MONTH-DAY-title.MARKUP
```

Например: `2024-01-15-getting-started.md`

Каждый пост начинается с front matter:

```yaml
---
layout: post
title: "Мой первый пост"
date: 2024-01-15
categories: [tutorial]
---
```
